# 野球チーム配車調整ボード

野球チーム向けに、日ごとの参加状況・同伴者数・車出し・空席数を入力し、乗車予定人数と乗車可能人数の過不足を即時集計する小規模Web MVPです。

実装は依存パッケージなしの軽量HTML/CSS/JavaScriptです。`npm install` はロックファイル確認のみで完了し、`npm run dev` でローカルサーバーを起動します。

## 起動方法

```bash
npm install
npm run dev
```

表示されたローカルURLをブラウザで開きます。

本番相当の静的ビルド確認:

```bash
npm run build
```

## 主な機能

- 上部カレンダーから日付を選択
- 日付ごとにイベント名、会場、選手別回答を保存
- 添付画像の選手マスタ（12名）を初期データとして登録
- 2026年5月の各日付の出席情報（○=参加、×=欠席、△=未定）を初期データとして反映
- 事前登録済み選手リストの行入力
- 選手ごとの参加状況: 参加 / 欠席 / 未定
- 選手以外の同伴者数
- 車出し有無、運転手人数、運転手を除いた乗車可能人数
- ダッシュボード上段: 参加選手 / 欠席選手 / 未定
- ダッシュボード下段: 配車数 / 運転手 / 乗車予定人数 / 乗車可人数 / 過不足
- 不足 / ちょうど / 余裕ありの視覚表示
- ヘッダーの「選手管理」から別画面で選手マスタを管理
- 選手マスタ画面で選手追加、名前・学年編集、削除、初期選手へのリセット
- localStorageへの自動保存と初期化

## 参加者情報自動取り込みについて

今回の修正で、参加者情報の自動取り込み機能はMVPから外しました。

理由:
- 日ごとのカレンダー管理と選手管理を優先
- 外部Google Apps Scriptページは公開設定、ログイン要否、CORS、HTML構造の違いにより安定取得しづらい
- 手元運用では、まず画面上で日ごとに直接入力する方が確実

## データ設計メモ

現在の保存先はlocalStorageです。状態は以下に分かれています。

- `players`: 選手マスタ
- `selectedDate`: 現在選択中の日付
- `visibleMonth`: 表示中のカレンダー月
- `events[date]`: 日付ごとのイベント情報と選手別回答

将来Google SheetsやFirebaseへ移行する場合は、`src/main.js` の `loadBoardState` / `saveBoardState` を置き換える想定です。

## QAチェックリスト

- `npm install` が成功する
- `npm run build` が成功する
- `npm test` が成功する
- 初期表示で2026年5月の選手マスタと出席情報が反映されている
- カレンダーで別日を選ぶと、その日専用の入力状態になる
- 参加を「参加」に切り替えると参加選手数と乗車予定人数が増える
- 欠席、未定の人数が上段ダッシュボードに反映される
- 同伴者数は参加選手の行だけ集計対象になり、車出しの運転手合計は同伴者合計から差し引かれる
- 車ありにすると運転手欄に1が入り、配車数と運転手合計が増える
- 車ありにして空席を入力すると乗車可人数が増える
- 空席が不足すると赤系、ちょうどなら緑系、余裕ありなら青系の集計表示になる
- ヘッダーの「選手管理」で選手マスタ画面へ移動できる
- 選手マスタ画面で選手を追加できる
- 選手マスタ画面で選手名・学年を編集できる
- 選手マスタ画面で選手を削除できる
- ヘッダーの「配車画面」で日別配車入力へ戻れる
- 画面を再読み込みしても日ごとの入力内容が保持される
- 初期化ボタンで入力内容がリセットされる
- スマートフォン幅でも各行の入力項目が縦に並び、操作できる
