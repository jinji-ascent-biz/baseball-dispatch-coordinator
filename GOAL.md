# /goal: 99_個人_001_配車システム

## Goal
野球チーム向けに、調整さんのように各個人が予定・配車情報を投稿でき、当日の必要乗車人数と車出し可能人数を自動集計して不足/余裕を一目で確認できる小規模Webツールを作る。

## Primary Users
- チーム代表/配車担当者
- 選手本人または保護者

## MVP Scope
1. イベント/日付ごとに配車調整ページを作成できる。
2. あらかじめ選手名が一覧で並んでいる。
3. 選手ごとに以下を入力できる。
   - 選手の参加有無
   - 選手以外の参加有無/人数（保護者・兄弟など）
   - 配車/車出し有無
   - 車出し有の場合、運転手以外の乗車可能人数
4. 集計欄に以下を表示する。
   - 参加選手数
   - 選手以外の参加人数合計
   - 車に乗る必要人数 = 参加選手数 + 選手以外の参加人数合計
   - 乗車可能人数 = 車出しありの乗車可能人数合計
   - 不足/ちょうど/余裕の判定
5. 既存のGoogle Apps Script月間ページから、その日参加できる選手情報を自動取得できるか調査し、可能なら実装する。
   - 対象URL: https://script.google.com/macros/s/AKfycbyvboWdEeQL2uO1uVK_0mV_zJ4nU91_0MBxzOhLJ9RwtDOMCFZLB_XBo-hmuziwAuHu/exec?page=monthly
   - 自動取得がCORS/認証/DOM制約で難しい場合は、手動入力/貼り付け導線を実装する。
6. スマホで入力しやすいレスポンシブUIにする。
7. データ保存はMVPではローカルストレージまたは軽量JSON保存でよい。ただし将来Google Sheets/Firebase等に移行しやすい構造にする。

## Non-goals
- 決済、ログイン、LINE通知、本番運用監視はMVP外。
- 個人情報を過度に扱わない。
- 外部サイトに対する無許可の書き込みやスクレイピング負荷の高い処理はしない。

## Acceptance Criteria
- `npm install` / `npm run dev` で起動できる。
- 選手一覧から入力すると集計が即時更新される。
- 座席不足時は赤系、ちょうど/余裕は分かりやすく表示される。
- 外部参加情報取得の可否がREADMEに明記される。
- 基本テストまたは最低限の手動QA手順が用意される。
- Claude Codeレビューに回せる状態で成果物・検証結果が整理される。
